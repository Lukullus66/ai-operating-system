# PROJECT.md

## Vision

Build modern, headless, AI-assisted digital products that are visually refined, technically maintainable and easy to evolve.

This framework is optimized for editorial websites, brand websites, marketing sites, headless CMS projects and selective commerce experiences.

## Target stack

- Figma for design and prototyping
- Ship Studio for initial code generation
- Next.js with App Router
- React
- TypeScript
- Sanity as headless CMS
- Vercel for hosting
- GitHub for version control
- Formspark or server-side form handling
- Claude Code and Codex for AI-assisted development

## Product principles

- Clear content hierarchy
- Strong typography
- Generous spacing
- Accessible interaction patterns
- Fast loading pages
- SEO-friendly information architecture
- Maintainable components
- Reusable content models

## Technical principles

- Server Components by default
- Client Components only when needed
- No unnecessary JavaScript
- Clear component boundaries
- Typed data contracts
- Content separated from presentation
- Documentation close to the code

## Definition of ready

A task is ready when:

- the goal is clear
- the user outcome is clear
- affected areas are known
- risks are understood
- acceptance criteria are defined

## Definition of done

A task is done when:

- the implementation works
- the change is reviewed
- accessibility has been considered
- responsiveness has been checked
- performance has been considered
- SEO impact has been checked
- documentation is updated if needed
