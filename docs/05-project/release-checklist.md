# Release Checklist

## Purpose

A release is ready when the product, content, infrastructure and rollback path are ready.

## Pre-release

- [ ] Build succeeds
- [ ] Type check succeeds
- [ ] Lint succeeds if configured
- [ ] Preview deployment reviewed
- [ ] Environment variables configured
- [ ] CMS dataset correct
- [ ] Production domain configured
- [ ] Redirects configured
- [ ] Forms tested
- [ ] Analytics tested if used

## Content

- [ ] Final copy reviewed
- [ ] Images optimized
- [ ] Alt text added
- [ ] SEO metadata complete
- [ ] Legal pages checked if relevant
- [ ] Navigation checked

## Technical SEO

- [ ] Sitemap works
- [ ] Robots file correct
- [ ] Canonicals correct
- [ ] Structured data valid if used
- [ ] 404 page exists
- [ ] Redirects tested

## Launch

- [ ] Deploy production
- [ ] Verify homepage
- [ ] Verify key landing pages
- [ ] Verify forms
- [ ] Verify CMS-powered pages
- [ ] Verify mobile
- [ ] Verify metadata

## Rollback

- [ ] Previous deployment known
- [ ] Rollback owner known
- [ ] DNS changes documented if applicable
