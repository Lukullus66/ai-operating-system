# QA Checklist

## Purpose

QA verifies that the product works for users, not only for developers.

## Functional checks

- [ ] Main navigation works
- [ ] Footer links work
- [ ] Forms submit correctly
- [ ] Success states work
- [ ] Error states work
- [ ] Required fields validate
- [ ] Dynamic pages render correctly
- [ ] CMS content displays correctly

## Responsive checks

Test at least:

- [ ] mobile small
- [ ] mobile large
- [ ] tablet
- [ ] laptop
- [ ] desktop wide

Check:

- [ ] no horizontal overflow
- [ ] readable headings
- [ ] readable body text
- [ ] usable forms
- [ ] correct card stacking
- [ ] navigation usability

## Accessibility checks

- [ ] keyboard navigation
- [ ] visible focus states
- [ ] semantic headings
- [ ] alt text
- [ ] label association
- [ ] color contrast
- [ ] reduced motion behavior if animations exist

## SEO checks

- [ ] unique title
- [ ] meta description
- [ ] canonical URL
- [ ] Open Graph metadata
- [ ] sitemap
- [ ] robots rules
- [ ] redirects if applicable

## Performance checks

- [ ] optimized images
- [ ] optimized fonts
- [ ] no unnecessary client components
- [ ] no obvious layout shift
- [ ] third-party scripts reviewed
