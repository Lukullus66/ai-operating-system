# SEO Principles

## Goal

SEO must be preserved and improved during design, migration and development.

SEO is not a launch-day task.

## Core requirements

Every production site should define:

- crawlable URL structure
- unique meta titles
- useful meta descriptions
- canonical URLs
- Open Graph metadata
- sitemap
- robots rules
- structured data where relevant
- redirects for changed URLs

## Migration rule

When migrating from another platform, preserve existing successful URLs whenever possible.

If a URL changes, create a redirect.

## Content hierarchy

Use semantic HTML.

Every page should have:

- one meaningful H1
- logical heading order
- descriptive link text
- useful image alt text

## Technical checklist

Before launch:

- verify status codes
- test redirects
- check canonical URLs
- check indexability
- generate sitemap
- verify metadata
- check structured data
- check Core Web Vitals
